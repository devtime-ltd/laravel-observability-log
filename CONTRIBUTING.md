# Contributing

Contributions are welcome. Please open an issue before starting work on anything large or non-trivial so we can discuss the approach.

## Development

```bash
git clone https://github.com/devtime-ltd/laravel-observability-log.git
cd laravel-observability-log
composer install
composer test
```

## Pull requests

- Keep PRs focused on a single change.
- Add or update tests for any behaviour changes.
- Make sure `composer test` passes before submitting.

## AI usage

Using AI tools to assist with contributions is fine. However, you should be able to understand, communicate, and ultimately defend your contribution without AI assistance. If you can't explain why a change was made or how it works, it's not ready to submit.
